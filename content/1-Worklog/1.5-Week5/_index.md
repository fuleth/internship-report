* Understand the core security concepts and best practices in AWS.

* Explore AWS security services such as AWS KMS and AWS Security Hub to protect cloud resources.
* Gain hands-on experience implementing security controls, access management, and encryption through practical AWS labs.

### Week 5 Achievements:

* Developed a solid understanding of the AWS Shared Responsibility Model and learned how AWS Identity and Access Management (IAM) is used to securely manage users, groups, roles, and permissions.
* Explored Amazon Cognito for authentication and user management, AWS Organizations for multi-account governance, AWS IAM Identity Center for centralized access management, AWS Key Management Service (KMS) for encryption key management, and AWS Security Hub for centralized security monitoring.
* Successfully completed hands-on labs covering AWS Security Hub, IAM Roles, IAM Conditions, Permission Boundaries, AWS KMS, Resource Tags, and Resource Groups, strengthening practical knowledge of AWS security and governance.
* Gained practical experience in controlling access to AWS resources using IAM policies, resource tags, IAM roles, and permission boundaries while applying the principle of least privilege.
* Learned how to integrate AWS security services to improve resource protection, access control, and operational security across AWS environments.
